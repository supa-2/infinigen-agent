# 场景生成时间估算

## 时间对比

### 无家具场景（使用 `disable/no_objects`）
- **场景生成时间**: 约 **4-5 分钟**
- **渲染时间**: 约 **1-2 分钟**
- **总计**: 约 **5-7 分钟**

### 有家具场景（使用 `base` 配置）
- **场景生成时间**: 约 **8-15 分钟**
  - Terrain 阶段: 约 1-2 分钟
  - Room solving 阶段: 约 2-3 分钟
  - 家具生成和放置: 约 3-6 分钟（**主要耗时增加在这里**）
  - 材质和装饰: 约 2-4 分钟
- **渲染时间**: 约 **2-4 分钟**（场景更复杂）
- **总计**: 约 **10-19 分钟**

## 为什么有家具会更慢？

1. **对象数量增加**: 需要生成和放置大量家具对象（沙发、茶几、桌子、椅子等）
2. **约束求解**: 需要解决家具放置的约束问题（空间布局、碰撞检测等）
3. **材质应用**: 每个家具都需要应用材质和纹理
4. **场景复杂度**: 更复杂的场景需要更多的计算资源
