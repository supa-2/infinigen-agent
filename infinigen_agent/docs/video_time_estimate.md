# 视频渲染时间估算

## 单帧渲染时间
根据实际测试：
- **单帧渲染时间**: 约 12分10秒（730秒）
- **后处理时间**: 约 0.6秒
- **总计**: 约 **12分11秒/帧**

## 多帧视频时间估算

### 不同帧数的渲染时间：

| 帧数 | 时长（秒） | 时长（分钟） | 时长（小时） | 视频时长（24fps） |
|------|-----------|-------------|-------------|------------------|
| 10帧 | ~122分钟 | ~2小时 | ~0.03小时 | 0.4秒 |
| 30帧 | ~366分钟 | ~6小时 | ~0.1小时 | 1.25秒 |
| 60帧 | ~732分钟 | ~12小时 | ~0.5小时 | 2.5秒 |
| 120帧 | ~1464分钟 | ~24小时 | ~1天 | 5秒 |
| 240帧 | ~2928分钟 | ~48小时 | ~2天 | 10秒 |
| 480帧 | ~5856分钟 | ~96小时 | ~4天 | 20秒 |

## 建议

### 快速测试（推荐）
- **10帧** - 约2小时，适合测试流程
- **30帧** - 约6小时，适合短视频

### 生产使用
- **60帧** - 约12小时，2.5秒视频
- **120帧** - 约24小时，5秒视频（推荐）
- **240帧** - 约48小时，10秒视频

## 优化建议

1. **降低分辨率**: 使用较低分辨率可以显著减少渲染时间
   - 1280x720 比 1920x1080 快约30-40%
   
2. **减少采样数**: 降低 Cycles 渲染的采样数可以加快速度
   - 但会降低质量

3. **使用 GPU**: 确保使用 GPU 加速（OPTIX/CUDA）
   - 当前已使用 OPTIX

4. **批量渲染**: 可以并行渲染多帧（如果有多个GPU）

## 实际使用示例

```bash
# 快速测试（10帧，约2小时）
bash create_video_from_scene.sh 10 24

# 标准视频（60帧，约12小时）
bash create_video_from_scene.sh 60 24

# 高质量视频（120帧，约24小时）
bash create_video_from_scene.sh 120 24
```

## 注意事项

- 渲染时间会根据场景复杂度变化
- 建议先用少量帧测试（如10帧）
- 可以后台运行，使用 `nohup` 或 `screen`/`tmux`
- 定期检查进度和磁盘空间
